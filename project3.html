Description

Je devais construire un gant avec un capteur gyroscopique pour contrôler le mouvement des moteurs afin de tourner ou de déplacer le robot d'avant en arrière, et un capteur de pression pour fermer la pince du robot avec nos doigts. Par exemple, il peut manipuler des objets toxiques, des objets fondants, etc. avec ce type de système, mais pas de manière automatique. automatiquement. C'est une sorte d'aide ; imaginez que l'on mette cela sur un buggy pour déplacer un conteneur d'uranium endommagé ou une bombe. Le système peut être plus précis qu'un contrôleur de base et plus intuitif.

L'installation comprend :

Bras robotique:

Contrôlé par un microcontrôleur ESP32 utilisant un pilote PCA9685 pour gérer plusieurs servomoteurs. Il fonctionne sur la base de commandes envoyées par le biais d'une communication WebSocket.

Contrôleur de gants:

Équipé d'un capteur gyroscopique (GY-521) et d'un microcontrôleur ESP32 pour transmettre sans fil les mouvements de la main au bras robotique. Utilise WebSocket pour la communication.

Application mobile:

Construite avec React Native, l'application fournit une interface utilisateur pour contrôler le bras robotique par le biais de commandes WebSocket. Les fonctionnalités comprennent l'authentification de l'utilisateur et le contrôle en temps réel du bras.

Base de données et serveur:

Un Raspberry Pi 4 fait tourner une base de données MySQL et Portainer pour la gestion des conteneurs Docker. Les conteneurs Docker incluent MySQL et phpMyAdmin pour la gestion de la base de données. Node-RED, installé directement sur le Raspberry Pi, gère la communication série et stocke les journaux dans la base de données. Un serveur API fournit un accès à la base de données, permettant à l'application mobile de récupérer les données des journaux au format JSON.

L'ensemble du système intègre des composants matériels et logiciels pour créer un bras robotique polyvalent et télécommandé avec une gestion robuste des données et une communication en temps réel.
