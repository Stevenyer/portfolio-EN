Description

I intend to build a glove with a gyro sensor to control the movement of the motors to turn or move. forwards and backward, and a pressure sensor to close the robot's gripper with our fingers. For example, it can manipulate toxic objects, melting objects, etc. with this sort of system, but not automatically. That is a sort of help; imagine putting that on a buggy to move a damaged uranium container or bomb. can be more precise than a basic controller and more intuitive.

The setup includes:

Robotic Arm:

Controlled by an ESP32 microcontroller using a PCA9685 driver to manage multiple servo motors. Operates based on commands sent through WebSocket communication.

Glove Controller:

Equipped with a gyroscope sensor (GY-521) and an ESP32 microcontroller to transmit hand movements to the robotic arm wirelessly. Utilizes WebSocket for communication.

Mobile Application:

Built with React Native, the app provides a user interface to control the robotic arm through WebSocket commands. Features include user authentication and real-time control of the arm.

Database and Server:

A Raspberry Pi 4 runs a MySQL database and Portainer for Docker container management. Docker containers include MySQL and phpMyAdmin for database management. Node-RED, installed directly on the Raspberry Pi, handles serial communication and stores logs in the database. An API server provides access to the database, allowing the mobile app to retrieve log data in JSON format.

The entire system integrates hardware and software components to create a versatile and remote-controlled robotic arm with robust data management and real-time communication.
